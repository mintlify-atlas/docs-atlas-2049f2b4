## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Onchain Portfolio repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd onchain-portfolio
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required packages using npm:
    
    ```bash
    npm install
    ```
    
    This will install all dependencies listed in `package.json`, including:
    - Next.js 14.2.35
    - React 18.2.0
    - RainbowKit 2.2.0
    - wagmi 2.13.2
    - viem 2.21.52
    - And more...
  </Step>

  <Step title="Environment Setup">
    The application is pre-configured for Sepolia testnet with the deployed contract address:
    
    ```javascript src/app/constants.js
    export const CONTRACT_ADDRESS = "0x96127ffA4375Aee0acF87443d508f2F2bd3f4620";
    ```
    
    <Note>
      No `.env` file is required for basic usage. The contract address and network configuration are already set up.
    </Note>
  </Step>

  <Step title="Start Development Server">
    Run the development server:
    
    ```bash
    npm run dev
    ```
    
    The application will be available at [http://localhost:3000](http://localhost:3000)
    
    <Info>
      The app will automatically redirect from the homepage to `/portfolio`
    </Info>
  </Step>

  <Step title="Configure Your Wallet">
    In your Web3 wallet (MetaMask, etc.):
    
    1. Add Sepolia testnet if not already added
    2. Get free Sepolia ETH from a faucet
    3. Connect your wallet to the application
    
    <Warning>
      Make sure you're on **Sepolia Testnet**, not Ethereum mainnet. Transactions on mainnet will cost real ETH.
    </Warning>
  </Step>
</Steps>

## Key Dependencies Explained

<AccordionGroup>
  <Accordion title="RainbowKit (^2.2.0)" icon="wallet">
    Best-in-class wallet connection UI for React applications. Provides:
    - Multiple wallet support (MetaMask, Coinbase, WalletConnect, etc.)
    - Beautiful, customizable connection modal
    - Automatic wallet state management
    - Network switching capabilities
    
    ```typescript Usage in RainbowKitContext.js
    import { getDefaultConfig, RainbowKitProvider } from "@rainbow-me/rainbowkit";
    import "@rainbow-me/rainbowkit/styles.css";
    ```
  </Accordion>

  <Accordion title="wagmi (^2.13.2)" icon="react">
    React Hooks library for Ethereum. Provides hooks for:
    - Reading from smart contracts (`useReadContract`)
    - Writing to smart contracts (`useWriteContract`)
    - Account management (`useAccount`)
    - Network information
    
    ```typescript Example Usage
    import { useAccount, useReadContract, useWriteContract } from "wagmi";
    
    const { address } = useAccount();
    const { data } = useReadContract({
      address: CONTRACT_ADDRESS,
      abi,
      functionName: "getAllWork"
    });
    ```
  </Accordion>

  <Accordion title="viem (^2.21.52)" icon="code">
    TypeScript interface for Ethereum. Provides:
    - Type-safe contract interactions
    - ABI encoding/decoding
    - Transaction formatting
    - Low-level Ethereum utilities
    
    Works seamlessly with wagmi under the hood.
  </Accordion>

  <Accordion title="Next.js (14.2.35)" icon="next-js">
    React framework with:
    - App Router (used in this project)
    - Server and Client Components
    - Built-in routing
    - Optimized performance
    
    ```typescript App Structure
    src/app/
    ├── page.tsx (redirects to /portfolio)
    ├── layout.tsx (root layout with providers)
    ├── portfolio/page.js (portfolio view)
    └── form/
        └── experience/page.tsx (add experience form)
    ```
  </Accordion>

  <Accordion title="Tailwind CSS & shadcn/ui" icon="paintbrush">
    Utility-first CSS framework with pre-built components:
    - Responsive design utilities
    - Custom gradient classes
    - shadcn/ui components: Button, Card, Input, Textarea, Badge
    - Dark mode support with next-themes
    
    ```typescript Component Example
    import { Button } from "@/components/ui/button";
    import { Card, CardContent } from "@/components/ui/card";
    ```
  </Accordion>

  <Accordion title="TanStack Query (^5.61.5)" icon="database">
    Powerful data synchronization for React:
    - Automatic caching
    - Background refetching
    - Required by RainbowKit for optimal performance
    
    ```typescript Setup in RainbowKitContext.js
    import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
    
    const queryClient = new QueryClient();
    ```
  </Accordion>
</AccordionGroup>

## Configuration Files

### RainbowKit Configuration

The wallet connection is configured in `src/context/RainbowKitContext.js`:

```typescript src/context/RainbowKitContext.js
"use client";
import React, { createContext } from "react";
import "@rainbow-me/rainbowkit/styles.css";
import { getDefaultConfig, RainbowKitProvider } from "@rainbow-me/rainbowkit";
import { WagmiProvider } from "wagmi";
import { sepolia } from "wagmi/chains";
import { QueryClientProvider, QueryClient } from "@tanstack/react-query";

const config = getDefaultConfig({
  appName: "My RainbowKit App",
  projectId: "9b4a97010a2d03a05e71209f233ab457", // WalletConnect project ID
  chains: [sepolia],
  ssr: true, // Enable server-side rendering
});

export const RainbowKitContext = createContext();
const queryClient = new QueryClient();

export const RainbowProvider = ({ children }) => {
  return (
    <RainbowKitContext.Provider value={config}>
      <WagmiProvider config={config}>
        <QueryClientProvider client={queryClient}>
          <RainbowKitProvider>{children}</RainbowKitProvider>
        </QueryClientProvider>
      </WagmiProvider>
    </RainbowKitContext.Provider>
  );
};
```

### Root Layout

Providers are wrapped in `src/app/layout.tsx`:

```typescript src/app/layout.tsx
import { Inter } from "next/font/google";
import "./globals.css";
import { ThemeProvider } from "@/context/ThemeContext";
import { RainbowProvider } from "@/context/RainbowKitContext";

const inter = Inter({ subsets: ["latin"] });

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={inter.className}>
        <RainbowProvider>
          <ThemeProvider>
            <main className="min-h-screen bg-background text-foreground">
              {children}
            </main>
          </ThemeProvider>
        </RainbowProvider>
      </body>
    </html>
  );
}
```

## Troubleshooting Installation

<AccordionGroup>
  <Accordion title="Node version issues">
    Make sure you're using Node.js 18.0 or higher:
    
    ```bash
    node --version
    ```
    
    If needed, update Node.js or use [nvm](https://github.com/nvm-sh/nvm) to manage versions.
  </Accordion>

  <Accordion title="npm install failures">
    If you encounter errors during `npm install`:
    
    ```bash
    # Clear npm cache
    npm cache clean --force
    
    # Delete node_modules and package-lock.json
    rm -rf node_modules package-lock.json
    
    # Reinstall
    npm install
    ```
  </Accordion>

  <Accordion title="Webpack/Build errors">
    The project includes polyfills for Node.js modules in `next.config.js`. If you encounter build errors:
    
    1. Ensure all dependencies are installed
    2. Check that you're using compatible versions
    3. Try deleting `.next` folder and rebuilding:
    
    ```bash
    rm -rf .next
    npm run build
    ```
  </Accordion>

  <Accordion title="RainbowKit connection issues">
    If wallet connections aren't working:
    
    1. Check that RainbowKit styles are imported in `RainbowKitContext.js`
    2. Verify the WalletConnect project ID is valid
    3. Make sure providers are wrapped correctly in `layout.tsx`
    4. Check browser console for errors
  </Accordion>
</AccordionGroup>
