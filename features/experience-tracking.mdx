## Overview

Experience tracking is the core feature of Onchain Portfolio, allowing you to permanently record your professional work experiences on the Ethereum blockchain (Sepolia testnet).

<Info>
  Work experiences are stored **on-chain**, making them immutable, verifiable, and accessible from any device once you connect your wallet.
</Info>

## Adding Experiences

Experiences are added through a dedicated form at `/form/experience`:

### Experience Form Interface

```tsx src/app/form/experience/page.tsx
import { useWriteContract } from "wagmi";
import { CONTRACT_ADDRESS } from "../../constants";
import abi from "../../contract/abi.json";

export default function ExperienceForm() {
  const [pow, setpow] = useState({
    company: "",
    link: "",
    position: "",
    description: "",
  });

  const {
    data,
    error,
    writeContract: createUserWriteContract,
  } = useWriteContract();

  const createPOW = async () => {
    createUserWriteContract({
      address: CONTRACT_ADDRESS,
      abi,
      functionName: "addWork",
      args: [pow.position, pow.description, pow.link, pow.company],
    });
  };

  return (
    <form className="space-y-4">
      <Input
        value={pow.company}
        onChange={(e) => setpow({ ...pow, company: e.target.value })}
        placeholder="Company"
      />
      <Input
        value={pow.link}
        onChange={(e) => setpow({ ...pow, link: e.target.value })}
        placeholder="Link"
      />
      <Input
        value={pow.position}
        onChange={(e) => setpow({ ...pow, position: e.target.value })}
        placeholder="Position"
      />
      <Textarea
        value={pow.description}
        onChange={(e) => setpow({ ...pow, description: e.target.value })}
        placeholder="Description"
      />
      <Button type="button" onClick={createPOW}>
        Add Experience
      </Button>
    </form>
  );
}
```

### Form Fields

<ParamField path="company" type="string" required>
  The name of the company or organization where you worked
</ParamField>

<ParamField path="position" type="string" required>
  Your role or job title (e.g., "Senior Blockchain Developer")
</ParamField>

<ParamField path="description" type="string" required>
  A detailed description of your responsibilities, achievements, or the project you worked on
</ParamField>

<ParamField path="link" type="string" required>
  A URL linking to the project, company website, or proof of work
</ParamField>

### Smart Contract Function

The `addWork` function is called to store the experience:

```solidity
function addWork(
  string memory _role,
  string memory _description,
  string memory _link,
  string memory _company
) public
```

<Note>
  The function arguments in the contract are: `role`, `description`, `link`, `company` - in that order.
</Note>

  </Step>
</Steps>

The form implements robust transaction handling with success and error states:


### Success State

When the transaction succeeds:
- An alert notification appears: "pow created successfully"
- Transaction data is logged to console
- Experience is permanently stored on-chain
- You can view it immediately on your portfolio page

## Retrieving Experiences

Once added, experiences can be retrieved on the portfolio page:

```javascript src/app/portfolio/page.js
import { useReadContract } from "wagmi";

const { data: isAccountData, error: isAccountError } = useReadContract({
  abi,
  address: CONTRACT_ADDRESS,
  functionName: "getAllWork",
  args: [],
});

return (
  <div>
    {isAccountData?.map((exp, index) => (
      <div key={index}>
        <h3>{exp.role}</h3>
        <p>{exp.company}</p>
        <p>{exp.description}</p>
        <a href={exp.link}>View Project</a>
      </div>
    ))}
  </div>
);
```

### Data Structure

Each experience returned from the contract includes:

```typescript
interface ProofOfWork {
  role: string;        // Position/title
  description: string; // Work description
  link: string;        // Project URL
  company: string;     // Company name
  timestamp: uint256;  // Block timestamp (automatically added)
}
```

<Info>
  The `timestamp` field is automatically set by the smart contract when the experience is added, recording the exact time it was stored on-chain.
</Info>

## Code Example: Complete Flow

```typescript
import { useState, useEffect } from "react";
import { useWriteContract } from "wagmi";
import { CONTRACT_ADDRESS } from "../../constants";
import abi from "../../contract/abi.json";

export default function AddExperience() {
  const [experience, setExperience] = useState({
    company: "Acme Corp",
    position: "Senior Developer",
    description: "Led development of DeFi protocol",
    link: "https://github.com/acme/defi"
  });

  const { data, error, writeContract } = useWriteContract();

  const handleSubmit = async () => {
    try {
      writeContract({
        address: CONTRACT_ADDRESS,
        abi,
        functionName: "addWork",
        args: [
          experience.position,
          experience.description,
          experience.link,
          experience.company
        ],
      });
    } catch (err) {
      console.error("Submission error:", err);
    }
  };

  useEffect(() => {
    if (data) {
      console.log("✅ Experience added:", data);
    }
    if (error) {
      console.error("❌ Error:", error);
    }
  }, [data, error]);

  return (
    <button onClick={handleSubmit}>
      Add Experience
    </button>
  );
}
```
