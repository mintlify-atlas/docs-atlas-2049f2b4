The Onchain Portfolio uses a collection of accessible, customizable UI components built with Radix UI primitives and styled with Tailwind CSS.

## Card

A flexible container component for grouping related content.

### Components

- `Card` - Main container with rounded borders and shadow
- `CardHeader` - Header section with padding
- `CardTitle` - Styled title text
- `CardDescription` - Muted description text
- `CardContent` - Main content area
- `CardFooter` - Footer section for actions

### Usage Example

```tsx
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";

<Card className="glass">
  <CardHeader>
    <CardTitle className="text-2xl font-bold">
      Experience
    </CardTitle>
  </CardHeader>
  <CardContent>
    <p>Your content here</p>
  </CardContent>
</Card>
```

**Source:** `src/components/ui/card.tsx`

### Props

All Card components extend standard `HTMLDivElement` attributes and accept:

- `className` - Additional CSS classes for customization
- `children` - React nodes to render inside
- `ref` - Forward ref support for all components

## Button

A versatile button component with multiple variants and sizes.

### Variants

- `default` - Primary button with solid background
- `destructive` - For destructive actions (red)
- `outline` - Bordered button with transparent background
- `secondary` - Secondary style with muted colors
- `ghost` - Minimal button with hover effects
- `link` - Text button with underline

### Sizes

- `default` - Standard size (h-9)
- `sm` - Small size (h-8)
- `lg` - Large size (h-10)
- `icon` - Square button for icons (h-9 w-9)

### Usage Example

```tsx
import { Button } from "@/components/ui/button";

<Button 
  variant="default" 
  size="lg"
  onClick={() => console.log('clicked')}
>
  Add Experience
</Button>

<Button variant="ghost" size="icon">
  <Icon className="h-4 w-4" />
</Button>
```

**Source:** `src/components/ui/button.tsx:43-54`

### Props

```typescript
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'default' | 'destructive' | 'outline' | 'secondary' | 'ghost' | 'link'
  size?: 'default' | 'sm' | 'lg' | 'icon'
  asChild?: boolean // Render as a Slot component
}
```

## Theme Toggle

A specialized button component for switching between light and dark themes.

### Usage Example

```tsx
import { ThemeToggle } from "@/components/theme-toggle";

<header>
  <nav>
    {/* Other nav items */}
    <ThemeToggle />
  </nav>
</header>
```

**Source:** `src/app/form/experience/page.tsx:75`

### Implementation

The ThemeToggle component uses the `useTheme` hook from ThemeContext:

```tsx
import { useTheme } from '@/context/ThemeContext';
import { Button } from "@/components/ui/button";
import { Moon, Sun } from 'lucide-react';

export function ThemeToggle() {
  const { theme, toggleTheme } = useTheme();

  return (
    <Button variant="ghost" size="icon" onClick={toggleTheme}>
      {theme === 'light' ? <Sun /> : <Moon />}
      <span className="sr-only">Toggle theme</span>
    </Button>
  );
}
```

## Styling

All components use the `cn()` utility function for conditional class merging:

```typescript
import { cn } from "@/lib/utils";

// Merge classes conditionally
const className = cn(
  "base-classes",
  condition && "conditional-classes",
  customClassName
);
```

Components are fully customizable through the `className` prop and support Tailwind CSS classes.
