## Environment Variables

The application doesn't require environment variables for basic operation, but you may need them for:

- Custom RPC endpoints
- API keys for blockchain data providers
- Analytics or monitoring services

### Creating .env.local

Create a `.env.local` file in the project root:

```bash
# Optional: Custom RPC endpoint
NEXT_PUBLIC_RPC_URL=https://sepolia.infura.io/v3/YOUR_KEY

# Optional: WalletConnect Project ID
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=your_project_id

# Contract address (already in constants.js)
NEXT_PUBLIC_CONTRACT_ADDRESS=0x96127ffA4375Aee0acF87443d508f2F2bd3f4620
```

<Warning>
Never commit `.env.local` to version control. Add it to `.gitignore`.
</Warning>

### Dark Mode Strategy

The app uses **class-based dark mode**:

```html
<!-- Light mode -->
<html lang="en">
  <body>...</body>
</html>

<!-- Dark mode -->
<html lang="en" class="dark">
  <body>...</body>
</html>
```

### Theme Provider Configuration

**Location:** `src/context/ThemeContext.tsx:14-32`

```tsx
export const ThemeProvider = ({ children }: { children: React.ReactNode }) => {
  const [theme, setTheme] = useState('dark')

  const toggleTheme = () => {
    setTheme(prevTheme => prevTheme === 'dark' ? 'light' : 'dark')
  }

  useEffect(() => {
    document.documentElement.classList.toggle('dark', theme === 'dark')
  }, [theme])

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      <NextThemesProvider attribute="class" defaultTheme="dark" enableSystem>
        {children}
      </NextThemesProvider>
    </ThemeContext.Provider>
  )
}
```

**Configuration Options:**
- `attribute="class"` - Uses class-based dark mode
- `defaultTheme="dark"` - Starts in dark mode
- `enableSystem` - Respects OS preference

## Next.js Configuration

### Webpack Customization

**Location:** `next.config.js:3-34`

```javascript
const nextConfig = {
  webpack: (config, { isServer }) => {
    // Ignore .map and .d.ts files
    config.module.rules.push(
      {
        test: /\.map$/,
        loader: 'ignore-loader'
      },
      {
        test: /\.d\.ts$/,
        loader: 'ignore-loader'
      }
    );

    // Polyfills for Web3 libraries
    config.resolve.fallback = {
      ...config.resolve.fallback,
      fs: false,
      net: false,
      tls: false,
      crypto: require.resolve('crypto-browserify'),
      stream: require.resolve('stream-browserify'),
      url: require.resolve('url'),
      zlib: require.resolve('browserify-zlib'),
      http: require.resolve('stream-http'),
      https: require.resolve('https-browserify'),
      assert: require.resolve('assert'),
      os: require.resolve('os-browserify'),
      path: require.resolve('path-browserify'),
    };

    return config;
  },
  transpilePackages: ['@metamask/sdk']
}
```

**Why These Settings:**
- **Polyfills:** Web3 libraries require Node.js modules in browser
- **Ignore loaders:** Prevent build errors from source maps
- **transpilePackages:** MetaMask SDK needs transpilation

## Performance Optimization

### Image Optimization

Next.js automatically optimizes images:

```tsx
import Image from 'next/image';

<Image
  src="/avatar.png"
  alt="Avatar"
  width={48}
  height={48}
  priority // Load immediately
/>
```

### Font Optimization

**Location:** `src/app/layout.tsx:1`

```tsx
import { Inter } from "next/font/google";

const inter = Inter({ subsets: ["latin"] });
```

Next.js automatically:
- Downloads fonts at build time
- Self-hosts fonts (no external requests)
- Prevents layout shift

### Bundle Analysis

Analyze bundle size:

```bash
npm install @next/bundle-analyzer
```

Add to `next.config.js`:

```javascript
const withBundleAnalyzer = require('@next/bundle-analyzer')({
  enabled: process.env.ANALYZE === 'true',
})

module.exports = withBundleAnalyzer(nextConfig)
```

Run analysis:

```bash
ANALYZE=true npm run build
```
